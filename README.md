# 🌼 蒲公英 AI Vibe-Match 审美匹配引擎

**小红书商业化蒲公英平台选人能力升级 Demo —— 从“传统标签筛选”进化到“多模态自然语言审美匹配”。**

> ⚠️ **项目说明**：本项目为小红书 RPT 岗位笔试/面试专属 MVP Demo。旨在探讨 AI 技术如何赋能 B 端商业化系统，打破流量内卷。

---

## 💡 业务背景与用户痛点 (Why I build this)

在深度体验小红书商业化生态后，我发现目前 B 端商家（品牌方）在蒲公英平台进行达人投放时，面临三大核心痛点：

1. **流量极度内卷（红海困境）**：传统选人高度依赖“类目+标签”（如：美妆商家只能搜美妆博主），导致头部和垂类标签流量拥挤，**商家买量成本（CPE/CPA）居高不下**。
2. **缺乏跨界想象力（认知局限）**：许多品牌亟需通过跨界（如：家电跨界穿搭、美妆跨界艺术）来寻找蓝海增量人群，但**机器冷冰冰的标签无法量化“网感”与“氛围感（Vibe）”**，人工筛选成本极高。
3. **硬广破坏社区生态（体验降级）**：强行派发的垂类商单往往导致种草内容生硬、同质化，不仅博主接单意愿低，也损害了小红书“真实、有用”的社区底色。

## 🎯 产品解法与商业价值 (The Solution & Value)

为了破局，我设计并开发了这款 **AI Vibe-Match 审美跨界撮合引擎**。
它将商业化的匹配逻辑从**“以品找人”**升级为**“以审美找人”**。

- **对商家（降本增效）**：通过自然语言与多模态视觉特征，精准挖掘跨赛道的“高性价比蓝海 KOC”，大幅降低单次互动成本。
- **对博主（赋能创作）**：系统自动提供基于视觉对齐的“原生种草场景建议”，降低跨界接单的创作阻力。
- **对平台（生态守护）**：用“审美对齐”代替“强塞硬广”，让商业笔记更具网感，实现商业变现与用户体验的双赢。

---

## ✨ 核心功能亮点 (Features)

- 🗣️ **多模态自然语言选人**：抛弃繁琐的下拉菜单。商家只需上传商品原图 + 用大白话描述想要的“氛围感”，大模型即可提取视觉语义进行匹配。
- 📊 **Vibe Score 量化评分**：将感性的“网感”转化为理性的数据。从视觉调性、受众画像、情绪共鸣、场景契合、商业性价比五个维度，为商家提供确定的决策依据。
- 🎬 **原生种草场景生成 (Co-pilot)**：不仅推人，更推“创意”。为每位匹配到的博主自动生成量身定制的内容植入方案（如：将极简家电作为清冷感博主对镜自拍的背景）。
- 🎨 **一键看效果 (One-click Demo)**：内置 3 组针对小红书典型商业痛点的解决方案（高端护肤红海突围 / 中古家具低频渗透 / 中式养生心智重塑），点击即可极速体验 AI 匹配全流程。

## 🛠️ 技术架构与工程实现 (How it works)

本项目采用 `Python (Streamlit)` + `Coze (多模态 LLM Workflow)` 构建，并已完成云端部署。

```
1. Coze Workflow 搭建
   商品图（多模态输入） + 品牌需求文本
       ↓  Coze Workflow 编排
   大模型推理：理解品牌调性 → 跨品类博主匹配 → 生成推荐理由 & 种草场景
       ↓  输出结构化 JSON

2. 后端 API 对接（Python + Requests）
   通过 Coze /v1/workflow/run 接口发送请求，解析嵌套 JSON 响应
   图片先经 /v1/files/upload 上传获取 URL，再作为 workflow 参数传入
   异步加载：threading 后台调用 API，前端同步播放 loading 动画

3. 前端渲染（Streamlit + 自定义 CSS/SVG）
   Streamlit 构建页面布局（双栏、表单、按钮交互）
   自定义 CSS 覆盖默认样式，实现小红书视觉风格（圆角卡片、胶囊标签、品牌色）
   SVG 绘制半圆仪表盘，可视化 Vibe Score 及五维评分条

4. Mock 数据 & 降级策略
   3 组预设案例数据存储在 mock_data.json，点击案例按钮直接渲染，无需调用 API
   自定义输入走实时 API，失败时自动降级到预设数据

5. 部署（Streamlit Community Cloud）
   敏感凭证通过环境变量注入（python-dotenv 本地 / 平台 Secrets 线上）
   依赖通过 requirements.txt 声明，云端自动安装
```
